# Руководство по тестированию приложения Adventy

## Для HR-специалистов

Это руководство поможет вам протестировать приложение Adventy в рабочей среде (production).

---

## Что такое приложение Adventy?

Adventy - это календарь приключений, где каждый день с 10 декабря 2025 по 24 декабря 2025 года представлен карточкой. Пользователи могут открывать карточку текущего дня, чтобы увидеть задание на сегодня.

---

## Как открыть приложение?

1. Откройте браузер (Chrome, Firefox, Edge и т.д.)
2. Перейдите по адресу приложения: **http://81.17.154.37/**
3. Вы увидите календарь с карточками дат

**Производственный сервер:** http://81.17.154.37/

---

## Обычное использование

### Просмотр задания на сегодня

1. Найдите карточку с надписью **"Today"** (сегодняшняя дата выделена фиолетовым цветом)
2. Нажмите на эту карточку
3. Карточка откроется и покажет задание на сегодня
4. Чтобы закрыть карточку, нажмите на крестик (×) в правом верхнем углу или нажмите клавишу **Escape**

### Просмотр других дат

- Все даты отображаются в календаре, включая прошедшие и будущие
- Вы можете открыть карточку любой прошедшей даты и увидеть задание для этой даты
- Если вы попытаетесь открыть карточку будущей даты без специальных параметров, вы увидите сообщение: **"Обожди, не торопись, скоро новое приключение будет доступно!"**

## Тестирование с помощью специальных параметров

Для тестирования приложения вы можете использовать специальные параметры, которые позволяют просматривать задания для будущих дат и дат вне диапазона.

**Примечание:** Прошедшие даты можно открывать без каких-либо параметров - они доступны всем пользователям.

### Как использовать параметры?

Добавьте параметры в конец адреса сайта после знака `?`.

**Формат:**
```
http://81.17.154.37/?first=СЕКРЕТ1&second=СЕКРЕТ2
```

### Секретные значения для обхода валидации

Для тестирования используйте следующие секретные значения:

1. **Первый параметр (first)** - позволяет просматривать даты вне диапазона:
   ```
   CggAEEUYFhgeGDkyCggBEAAYgA_secret_to_skip_date_in_range
   ```

2. **Второй параметр (second)** - позволяет просматривать будущие даты:
   ```
   EgZjaHJvbWUyCggAE_secret_to_skip_date_has_not_appeared
   ```

## Сценарии тестирования

### Сценарий 1: Просмотр задания для прошедшей даты

**Цель:** Проверить, что можно открыть задание для даты, которая уже прошла.

**Шаги:**
1. Откройте браузер
2. Перейдите по адресу приложения (без параметров):
   ```
   http://81.17.154.37/
   ```
3. Найдите любую карточку с прошедшей датой (дата будет серого/синего цвета)
4. Нажмите на карточку
5. Карточка должна открыться и показать задание для этой даты

**Ожидаемый результат:**
- Прошедшие даты отображаются в календаре (все даты всегда видны)
- Карточка открывается и показывает задание для прошедшей даты без необходимости использования каких-либо параметров

---

### Сценарий 2: Просмотр задания для будущей даты

**Цель:** Проверить, что можно открыть задание для даты, которая еще не наступила.

**Шаги:**
1. Откройте браузер
2. Введите адрес с параметром `second`:
   ```
   http://81.17.154.37/?second=EgZjaHJvbWUyCggAE_secret_to_skip_date_has_not_appeared
   ```
3. Нажмите Enter
4. Найдите любую карточку с будущей датой
5. Нажмите на карточку
6. Карточка должна открыться и показать задание для этой даты

**Ожидаемый результат:** Карточка открывается и показывает задание, даже если дата еще не наступила.

---

### Сценарий 3: Просмотр задания для даты вне диапазона

**Цель:** Проверить работу с датами, которые находятся вне диапазона 10-31 декабря 2025.

**Шаги:**
1. Откройте браузер
2. Введите адрес с параметром `first`:
   ```
   http://81.17.154.37/?first=CggAEEUYFhgeGDkyCggBEAAYgA_secret_to_skip_date_in_range
   ```
3. Нажмите Enter
4. Попробуйте открыть любую карточку

**Ожидаемый результат:** Приложение работает нормально, даже если дата находится вне обычного диапазона.

---

### Сценарий 4: Комбинированное тестирование

**Цель:** Использовать несколько параметров одновременно для полного тестирования.

**Шаги:**
1. Откройте браузер
2. Введите адрес с параметрами для будущих дат и дат вне диапазона:
   ```
   http://81.17.154.37/?first=CggAEEUYFhgeGDkyCggBEAAYgA_secret_to_skip_date_in_range&second=EgZjaHJvbWUyCggAE_secret_to_skip_date_has_not_appeared
   ```
3. Нажмите Enter
4. Теперь вы можете:
   - Видеть все даты, включая прошедшие (все даты всегда отображаются)
   - Открывать карточки прошедших дат без параметров
   - Открывать карточки будущих дат и дат вне диапазона (благодаря секретным параметрам)

**Ожидаемый результат:**
- Все даты отображаются в календаре
- Все карточки открываются и показывают задания, независимо от даты

## Важные замечания

### Безопасность

⚠️ **ВАЖНО:** Эти параметры предназначены только для тестирования! Не делитесь этими секретными значениями с обычными пользователями.

### Когда использовать параметры?

Используйте параметры только когда нужно:
- Проверить задания для будущих дат
- Протестировать работу приложения с датами вне диапазона

**Примечание:** Прошедшие даты доступны всем пользователям без параметров.

### Обычные пользователи

Обычные пользователи должны использовать приложение **без параметров**. Они:
- Увидят все даты в календаре, включая прошедшие и будущие
- Смогут открывать карточки прошедших дат и текущего дня
- Для будущих дат будут показаны соответствующие сообщения

## Часто задаваемые вопросы

### Вопрос: Что делать, если карточка не открывается?

**Ответ:**
1. Убедитесь, что вы используете правильный секретный код
2. Проверьте, что адрес введен правильно (без пробелов)
3. Попробуйте обновить страницу (F5 или Ctrl+R)

### Вопрос: Можно ли использовать параметры на мобильном телефоне?

**Ответ:** Да, параметры работают на всех устройствах. Просто введите полный адрес с параметрами в адресной строке браузера.

### Вопрос: Нужно ли вводить параметры для просмотра прошедших дат?

**Ответ:** Нет, прошедшие даты можно открывать без каких-либо параметров. Параметры нужны только для просмотра будущих дат и дат вне диапазона.

### Вопрос: Что делать, если я вижу ошибку?

**Ответ:**
1. Проверьте правильность секретных кодов
2. Убедитесь, что дата существует в системе
3. Свяжитесь с технической поддержкой, если проблема сохраняется
